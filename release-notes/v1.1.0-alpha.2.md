# pdfnative v1.1.0-alpha.2

**Released:** 2026-04-29 · **Type:** alpha (pre-release) · **Git tag:** none yet (alpha)

This is the second alpha iteration on the path to **v1.1.0 stable**. It extends
alpha.1 with two table-layout features that were on the medium-term list, plus
a small UX polish to the documentation site (live `pdfnative-mcp` badge + a
compact version strip directly under the main navigation).

The two large epics — issue [#28](https://github.com/Nizoka/pdfnative/issues/28)
(PDF/A Latin font embedding) and issue [#25](https://github.com/Nizoka/pdfnative/issues/25)
(full UAX #9 BiDi + multi-pass GSUB + GPOS MarkBasePos), plus the new **emoji
monochrome support** committed for v1.1.0 — remain scheduled for the stable
cut. True page-by-page constant-memory streaming is deferred to v1.2.0 because
it requires an architectural refactor of `pdf-document.ts` that does not fit
the alpha-velocity envelope.

## Highlights

### Cell clipping (default)

`TableBlock.clipCells` (defaults to `true`) wraps every header and data cell
in a PDF clip-path rectangle (`q <rect> re W n ... Q`, ISO 32000-1 §8.5.4).
Content can no longer escape its column visually — even with proportional
fonts, mixed scripts, or aggressive sizes. Opt out with `clipCells: false`
to recover byte-identical v1.0.x output.

```ts
import { buildDocumentPDF } from 'pdfnative';

const pdf = buildDocumentPDF({
    title: 'Statement',
    blocks: [{
        type: 'table',
        headers: ['Date', 'Description', 'Amount'],
        rows: [
            // 'Description' is too long, but the column won't overflow.
            { cells: ['2026-01-15', 'A very long description …', '€1,234.56'], type: 'debit', pointed: false },
        ],
        // clipCells: true is the default; set false to opt out.
    }],
    footerText: 'Generated by pdfnative',
});
```

### Auto-fit column widths

`TableBlock.autoFitColumns: true` derives column-width fractions from
**actual measured content widths** (header at `fs.th`, cells at `fs.td`,
plus 6 pt cell padding). Per-column `minWidth` / `maxWidth` constraints
introduced in alpha.1 are still honoured by the downstream positioner.

```ts
const pdf = buildDocumentPDF({
    title: 'Auto-sized table',
    blocks: [{
        type: 'table',
        headers: ['ID', 'Long header text indeed', 'Total'],
        rows: rowsFromDb,
        autoFitColumns: true, // sizes columns to content
        columns: [
            { f: 0.2, a: 'l', mx: 8, mxH: 8, minWidth: 30 },
            { f: 0.6, a: 'l', mx: 30, mxH: 30 },
            { f: 0.2, a: 'r', mx: 12, mxH: 12, minWidth: 60 },
        ],
    }],
    footerText: 'pdfnative',
});
```

Defaults to `false` because byte-output depends on table content and font
metrics — only enable when content-aware sizing is desired.

### Documentation site UX polish

- New live **`pdfnative-mcp` npm version badge** in the hero badge strip,
  matching the existing `pdfnative-cli` badge.
- New **compact version strip** mounted directly under the main `<nav>`:
  one line, 11 pt, sticky-discreet, gives visitors immediate visibility
  into the current published `pdfnative` / `pdfnative-cli` /
  `pdfnative-mcp` versions and their transitive `pdfnative` pins.
- The richer **detailed widget** (footer block) is preserved verbatim.
- The strip is propagated to `docs/playgrounds/cli.html` and
  `docs/playgrounds/mcp.html`.

The widget is rendered by [docs/assets/versions.js](../docs/assets/versions.js),
which now auto-discovers all mount points
(`#pdfnative-versions`, `.pn-version-strip`, `[data-pn-versions]`) and picks
between `compact` and `detailed` rendering via the `data-mode` attribute or
the host class.

## Migration

`v1.0.5 → 1.1.0-alpha.2` is purely additive. Existing code paths are byte-stable
**unless** a table opts into `clipCells` (now defaults to `true` — visual
clipping but no API change) or `autoFitColumns: true` (opt-in). Set
`clipCells: false` on every `TableBlock` if exact v1.0.x byte-output is
required.
